### Step 3: IBM FlashSystem C200 storage – configure multipath I/O

Complete the steps to enable and configure multipathing for disk storage. For detailed instructions, see the documentation that is provided with your hardware.

**Procedure**

1. **Enable DM‑multipath** 
   * Install the `device-mapper-multipath` package if not already present.  
   * Run `mpathconf` to enable the service.
     ```
     mpathconf --enable --with_multipathd y --find_multipaths y
     ```

2. **Edit `/etc/multipath.conf`** 
   * Add a `defaults` stanza with `user_friendly_names yes` and `fast_io_fail_tmo 5`.  
   * Increase the SCSI timeout by adding a udev rule if recommended by IBM support.

3. **Adjust udev rules** 
   * Create a file `/etc/udev/rules.d/99-flash.rules` to set the SCSI timeout to 120 seconds:
     ```
     ACTION=="add|change", SUBSYSTEM=="scsi", ATTR{timeout}="120"
     ```

4. **Reload multipath** 
   * Run 
     ```
     systemctl restart multipathd
     ```
   * Verify that each LUN appears with two active paths, run:
     ```
     multipath -ll
     ```

5. **Label LUNs** 
   * Use the `alias` directive in `/etc/multipath.conf` to assign friendly names to each FlashSystem C200 LUN 
     (for example, `alias db_lun00`).  These names will be used in the YAML response file and mount commands.
